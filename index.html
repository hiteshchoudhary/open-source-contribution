Understanding Git and GitHub

Git is a version control system that allows you to track changes in your code over time, while GitHub is a platform that hosts your Git repositories online, enabling collaboration with others.

Key Git Concepts

  • Repository: A project folder containing all your code files and their version history.
  • Working Directory: The current set of files you are actively editing on your local machine.
  • Staging Area: A temporary holding area where you mark files to be included in your next commit.
  • Commit: A snapshot of your project at a specific point in time, with a descriptive message.
  • Branch: A parallel line of development, allowing you to work on different features without affecting the main codebase.

Basic Git Commands

Initializing a Git Repository:

git init

Adding Files to Staging Area:

git add

Committing Changes:

git commit -m "Commit message"

Using GitHub

Creating a Repository on GitHub:

  1. Go to GitHub and create a new repository.
  2. Copy the repository URL.

Connecting Local Repository to GitHub:

git remote add origin

Pushing Changes to GitHub:

git push -u origin master
